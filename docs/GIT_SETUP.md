# Настройка Git для проекта

## Структура проекта для Git

### Игнорируемые файлы и папки (.gitignore)

Проект настроен для исключения следующих элементов:

#### Данные и датасеты:
- `MINI-RGBD_web/` - полный датасет (слишком большой для git)
- `test_videos/` - тестовые видео
- `uploads/` - загруженные пользователями файлы

#### Результаты и выходные данные:
- `results/` - все результаты анализа
- `*.mp4`, `*.avi`, `*.mov`, `*.mkv` - видео файлы
- `*.png`, `*.jpg`, `*.jpeg` - изображения
- `*.json` - результаты анализа (кроме config.json)

#### Модели и чекпоинты:
- `checkpoints/*.pt` - файлы моделей (слишком большие)
- `checkpoints/*.pth` - альтернативный формат моделей
- Сохранена структура папки через `.gitkeep`

#### Python:
- `__pycache__/` - кэш Python
- `*.pyc`, `*.pyo` - скомпилированные файлы
- `venv/`, `.venv/` - виртуальные окружения

#### IDE и временные файлы:
- `.vscode/`, `.idea/` - настройки IDE
- `*.log` - логи
- `*.tmp`, `*.temp` - временные файлы

## Что будет в репозитории

### Код:
- ✅ Все `.py` файлы
- ✅ `config.yaml` - конфигурация
- ✅ `requirements.txt` - зависимости

### Документация:
- ✅ `README.md` - основная документация
- ✅ `docs/` - дополнительная документация
  - `docs/USAGE.md` - инструкция по использованию
  - `docs/archive/` - архивная документация

### Структура:
- ✅ `models/` - код моделей
- ✅ `utils/` - утилиты
- ✅ `checkpoints/.gitkeep` - структура папки (без файлов)

## Первоначальная настройка Git

```bash
# Инициализация репозитория (если еще не инициализирован)
git init

# Добавление всех файлов (с учетом .gitignore)
git add .

# Первый коммит
git commit -m "Initial commit: Infant motion anomaly detection system"

# Добавление remote (если есть)
git remote add origin <repository-url>

# Push в репозиторий
git push -u origin main
```

## Рекомендации

### Для работы с датасетом:
Датасет MINI-RGBD должен быть скачан отдельно и помещен в `MINI-RGBD_web/` после клонирования репозитория.

### Для работы с моделями:
Модели должны быть обучены локально или скачаны отдельно и помещены в `checkpoints/`.

### Для тестирования:
Тестовые видео должны быть помещены в `test_videos/` после клонирования.

## Размер репозитория

Без датасета и моделей репозиторий будет занимать примерно:
- Код: ~500 KB
- Документация: ~100 KB
- **Итого: ~600 KB**

С датасетом и моделями:
- Датасет MINI-RGBD: ~10-20 GB
- Модели: ~50-100 MB
- **Итого: ~10-20 GB** (не рекомендуется для git)
